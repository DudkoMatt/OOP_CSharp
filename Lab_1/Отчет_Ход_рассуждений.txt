В ходе написания данной лабораторной работы было создано 3 файла с исходным кодом на языке C#.

Первый - Program.cs
В данном файле находится класс с функцией Main

Второй - IniParserData.cs
В данном файле реализован класс, который хранит данные считанного файла .ini и позволяет обращаться к значениям полей, через методы.
Хранение данных файла .ini реализовано при помощи словаря словарей. Первый словарь, хранит имя секции [строка] и словарь, относящийся к данной секции. Внутренние словари хранят название поля [строка] и его значение. Значение поля также хранится в виде строки.
Добавление полей в данный класс реализовано при помощи функции AddField с соответствующей обработкой возможных ошибок (поле уже было добавлено ранее, секция уже была объявлена ранее и прочие). Если произойдет ошибка, то будет вызвано исключение FormatException.
Обращение к полям производится через методы TryGet. На вход методы принимают имя секции, имя поля, а также выходной параметр. Сами методы возвращают значение bool, которое показывает, были ли успешно произведены конвертирование значения поля из строки в тип выходного параметра и запись этого значения в выходной параметр.

Третий - IniParser.cs
В последнем файле реализован статический класс IniParser с методом Parse, который считывает данные из указанного файла и возвращает данные в виде экземпляра класса IniParserData.
Метод Parse принимает на вход путь до входного файла. В процессе выполнения он создает экземпляр класса IniParserData, считывает данные из файла построчно. При считывании очередной строки из нее удаляются комментарии, а также ведущие и последние пробелы. Если строка осталось пустой, то она пропускается. Далее идет обработка полученной строки: идет попытка распознать при помощи регулярных выражений является ли строка названием секции или парой название поля - значение. Если она является названием секции, то идет проверка того, что секция не повторялась. Если является парой название поля - значение, то производится попытка ее добавления в IniParserData. Если строка не является одним из двух вариантов или проверка секции или добавление поля произошло неудачно, то будет вызвано исключение FormatException.

В функции Main приведены примеры использования данных классов. Все исходные файлы, а также пример .ini файла приложены к лабораторной работе.
